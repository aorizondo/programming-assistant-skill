1. SOLUTION.md
   Open ChatGPT and tell it:

I am building a xxx (detailed description of your product - features, target users, key characteristics, the more detailed the better).
Frontend uses Vue.js, database uses PostgreSQL, backend uses Golang 1.22+.
Please help me generate a complete architecture design, including:

- File and folder structures
- The role of each part
- State storage location
- How services connect to each other
  Please output the entire document in markdown format.

Save ChatGPT's output as SOLUTION.md and put it in an empty folder, which will serve as your project root directory.

------------------------

2. TASK.md
   Tell ChatGPT 4o:

Based on the architecture in SOLUTION.md, write a detailed step-by-step build plan.
Each task should:

- Be accurate, concise, and testable
- Have clear start and end points
- Focus on a single issue
- Be ordered by implementation sequence

I will hand these tasks to an engineering LLM, executing one task at a time, being able to test each task unit.

Save the output as TASK.md in the project folder.

------------------------

3. Back in Cursor, tell it:

You are a senior software engineer developing this software project.
You have been provided with SOLUTION.md and TASK.md.

**Development Requirements:**

- Carefully read these two files to ensure no misunderstanding of what needs to be built.

- Follow the order in TASK.md, completing only one task at a time.

- Test after completing each task. If the test passes, proceed to the next task.

- Do not use emojis in code or documentation; minimize code comments (do not write unless essential); minimize the number of documents, keeping only main docs like README, SOLUTION, TASK, DEPLOYMENT.

**Development Rules:**

- Strictly use minimal code to complete the current task.

- Do not perform large-scale changes.

- Do not make irrelevant edits, focus solely on the task you are developing.

- Code must be precise, modular, and testable.

- Do not break existing functionality.
